# MintHive 兴趣社交平台开发软件需求规格说明书（SRS）

# 1 文档概述

## 1\.1 文档目的

本文档为**兴趣社交平台**唯一官方软件开发需求说明书，明确平台业务目标、用户角色、功能需求、非功能需求、数据库设计、接口规范、部署要求及验收标准。本文档作为产品设计、前端开发、后端开发、测试、运维及项目验收的核心依据，所有开发工作必须严格遵循本文档要求，无需求变更说明不得擅自修改功能逻辑。

## 1\.2 项目背景

当前主流社交平台泛社交属性过重，用户难以精准找到同圈层兴趣好友，垂直兴趣交流缺失轻量化、实时化互动能力。本平台聚焦**垂直兴趣圈层社交**，融合论坛图文发帖、圈子社群、实时评论互动、兴趣匹配、动态信息流五大核心能力，区别于通用社交软件，主打「以兴趣聚合人群，以内容产生互动」，支持用户自由发布兴趣内容、加入专属兴趣圈子、实时交流讨论、点赞收藏互动，打造轻量化、高粘性、垂直化的兴趣交流社区。

## 1\.3 平台定位

- **产品类型**：Web端\+PC管理后台\+移动端H5（后期可拓展小程序/APP）兴趣社交论坛平台

- **核心人群**：16\-40岁拥有垂直兴趣爱好的互联网用户（二次元、摄影、读书、游戏、露营、手工、影视等圈层）

- **项目英文名称**：MintHive（最终定名）

- **名称释义**：Mint代表新鲜志趣、年轻鲜活的兴趣爱好；Hive意为蜂巢，寓意同兴趣爱好者相聚成团、圈层聚集，完美契合平台圈子社交、同好聚合的核心定位，简洁好记、域名友好、无负面歧义，适合社交产品品牌传播。

## 1\.4 术语与缩略语

|术语|释义|
|---|---|
|动态|用户发布的单条图文/视频帖子，平台核心内容单元|
|兴趣圈子|基于统一兴趣建立的社群板块，分为公开圈、私密圈|
|实时互动|评论、回复、点赞、私信无需刷新页面，前端实时推送数据|
|风控审核|机器初审\+人工复审，拦截违规文案、图片、低俗内容|
|WebSocket|长连接协议，用于实现平台实时消息、实时评论推送|

# 2 整体业务概述

## 2\.1 整体业务总流程（含AI全链路嵌入）

在原有基础社交业务流程上，全链路内嵌AI智能能力，完整闭环业务流程如下：

1. 用户注册登录→完善初始兴趣标签→AI采集用户基础画像

2. 用户发帖：手动编辑文案/AI一键生成文案→AI前置内容预检→本地MinIO存储图片视频→AI\+敏感词双重风控审核→帖子发布上线

3. 内容浏览：AI千人千面兴趣信息流推荐→用户互动（点赞/评论）→手动评论/AI智能一键评论

4. 离线私信：用户离线时AI智能代回复私信→用户上线查看AI聊天记录，可撤回AI消息

5. 社区风控：全量用户图文、评论、私信经过AI语义复审\+本地敏感词初审，拦截隐性违规内容

6. 举报工单：AI自动分拣举报风险等级→高危工单优先推送管理员人工复核

7. 平台运营：AI每日自动生成社区健康度报表，辅助管理员运营决策

# 3 功能性需求

## 3\.0 平台新增AI智能辅助功能（核心增值模块）

依托大模型能力，为兴趣社交平台全链路植入AI功能，降低用户发帖门槛、优化社区内容质量、强化兴趣推荐精准度，同时支持**两种AI部署模式：公有云API调用 / 本地私有化AI部署**，适配项目本地服务器无外网、内网私有化部署需求，贴合本项目本地文件存储整体架构。

### 3\.0\.1 AI发帖助手（用户端高频核心AI功能）

1. **AI一键生成帖子文案**：用户上传图片/视频后，输入简短关键词（如：露营日落、新番吐槽、耳机测评），AI自动匹配对应兴趣圈层风格，生成3版不同风格文案（简约版、氛围感版、详细干货版），用户可直接选用、二次编辑后发布。

2. **AI文案润色/纠错**：用户写完草稿文案后，一键AI润色，优化语句通顺度、补充话题标签、修正错别字语病，同时贴合社区社交口语化风格，不生硬刻板。

3. **AI敏感内容预检测**：发帖前AI前置检测文案、图片隐性违规内容，提前提醒用户修改，减少帖子审核驳回率，补足原有机器风控短板。

### 3\.0\.2 AI智能互动功能

1. **AI智能评论回复**：用户浏览帖子无灵感评论时，点击AI评论按钮，AI根据帖子内容、所属兴趣圈子，自动生成贴合内容的真实自然评论，避免模板化水军评论；同时支持楼中楼AI一键回复他人评论。

2. **AI私信智能应答**：用户离线期间，AI可根据聊天上下文，自动回复好友基础私信消息，避免消息无人应答；用户上线后可查看全部AI代聊记录，支持一键撤回AI回复。

### 3\.0\.3 AI内容智能风控（双重审核，强化社区安全）

1. 原有敏感词库\+机器规则初审 \+ **AI大模型语义复审**：传统敏感词只能拦截固定关键词，AI可识别谐音、变体、暗语、阴阳怪气、人身攻击等隐性违规言论，全方位拦截软色情、引战、广告引流内容。

2. AI图片违规识别：识别图片内低俗、水印广告、二维码引流、违规截图内容，配合本地文件存储，图片上传后本地AI直接检测，无需外传图片至公网，保护用户隐私。

### 3\.0\.4 AI精准兴趣推荐（优化首页信息流核心体验）

1. 基于用户浏览记录、点赞/评论/发帖行为、停留时长，AI深度学习用户兴趣偏好，动态更新个人兴趣标签，淘汰固定标签模式，实现千人千面信息流推荐。

2. AI圈子智能推荐：自动推荐用户潜在感兴趣、但未加入的圈子，提升圈子入驻率与平台活跃度。

3. AI相似好友匹配：根据内容喜好、圈子参与情况，智能推荐同兴趣好友，强化平台社交匹配能力。

### 3\.0\.5 AI图片智能处理（适配本地图片存储）

1. AI图片美颜、画质修复、背景消除：用户上传本地图片发帖前，内置简易AI修图功能，所有图片处理全程在本地服务器完成，不调用外网图片接口，匹配项目本地文件存储方案。

2. AI图片智能裁剪：自动适配平台帖子图片尺寸，统一页面展示样式。

### 3\.0\.6 AI社区问答助手

全局悬浮AI客服入口，用户可咨询平台使用规则、圈子玩法、账号处罚问题，7\*24小时AI自动应答，复杂问题自动转接人工管理员后台。

### 3\.0\.7 管理员后台AI辅助能力

1. AI举报工单智能分拣：自动识别举报工单严重等级，优先推送暴力、色情高危工单给管理员，降低人工审核工作量。

2. AI内容复盘：每日自动生成社区内容健康度报告，统计违规内容类型、高发时段，给出社区运营优化建议。

1. 用户注册登录→完善个人兴趣标签→平台智能推荐兴趣圈子与内容

2. 用户浏览首页信息流/圈子内容→发布动态/评论/点赞/转发互动

3. 用户添加好友、发送私信、接收实时互动消息

4. 管理员后台审核内容、管理用户、维护圈子、处理举报投诉

5. 系统自动统计平台数据，生成用户活跃度、内容发布量报表

## 2\.2 用户角色划分

平台一共分为三类核心角色，权限逐级递增，具体权限划分如下：

### 2\.2\.1 普通注册用户（C端用户）

平台基础使用者，可浏览内容、发布动态、加入圈子、互动评论、私信好友、举报违规内容，无后台管理权限。

### 2\.2\.2 圈子管理员（圈主）

由普通用户申请、平台运营审核通过后开通权限，负责管理对应兴趣圈子：审核圈子入圈申请、删除圈内违规帖子、封禁圈内违规用户、置顶优质圈内内容。

### 2\.2\.3 平台超级管理员（后台运营）

拥有平台全部权限，管理全量用户、审核全部内容、创建/下架兴趣圈子、处理全站举报、配置系统参数、查看平台运营数据、封禁违规账号。

# 3 功能性需求

## 3\.1 前端用户端功能（Web\+H5）

### 3\.1\.1 账号登录注册模块

1. **注册功能**：支持手机号验证码注册、账号密码注册两种方式；注册强制勾选用户协议与隐私政策；注册成功后引导用户选择个人兴趣标签（最少3个，最多8个），用于后续内容推荐。

2. **登录功能**：账号密码登录、手机验证码登录、记住登录状态；支持忘记密码，通过手机验证码重置密码。

3. **第三方快捷登录（可选）**：微信快捷登录，适配移动端H5场景。

4. **个人中心**：修改头像、昵称、个人简介、性别、生日、兴趣标签；查看自己发布的动态、收藏、点赞、关注、粉丝列表；支持账号注销功能（7天冷静期，注销后清空所有个人数据）。

### 3\.1\.2 首页信息流模块（核心）

1. **信息流推荐逻辑**：默认按照兴趣标签智能推荐内容，其次展示关注用户动态、热门全站动态；支持切换推荐模式：智能推荐、最新发布、最热榜单。

2. **动态卡片展示**：单条动态展示发布人头像昵称、发布时间、文案内容、图片/视频、点赞数、评论数、转发数；支持下拉刷新、上拉加载更多。

3. **基础互动操作**：实时点赞/取消点赞、收藏/取消收藏、转发动态、查看全部评论；所有互动操作无需页面刷新，实时变更数据。

### 3\.1\.3 动态发布模块（核心发帖功能）

1. 支持发布纯文字、图文（最多9张图片）、短视频（时长≤60s）三种类型动态；

2. 发布动态可添加话题标签、选择可见范围：公开可见、仅粉丝可见、仅自己可见；

3. 发布内容自动进入机器风控审核，含违规内容直接拦截并提示用户，无违规内容直接发布；

4. 支持草稿箱功能，未编辑完成的动态自动保存草稿，下次进入发布页可继续编辑。

### 3\.1\.4 评论与回复模块（实时互动核心）

1. 支持一级评论、楼中楼二级回复，完整论坛层级评论结构；

2. 基于WebSocket实现**实时评论推送**：帖子有新评论时，无需刷新页面，自动弹窗提示并更新评论列表；

3. 支持评论点赞、删除本人评论；用户可举报违规评论；

4. 评论支持图片\+文字混合回复，适配论坛交流场景。

### 3\.1\.5 兴趣圈子模块（社群核心）

1. **圈子广场**：展示全部公开兴趣圈子，支持关键词搜索圈子、按分类筛选圈子（游戏、影视、户外、读书等）；展示圈子人数、圈子简介、圈内热门动态。

2. **入圈机制**：公开圈子一键直接加入；私密圈子需要填写申请理由，等待圈主审核通过后方可入圈；用户可同时加入最多20个圈子。

3. **圈子主页**：圈内专属信息流，仅展示圈内用户发布的动态；圈主可置顶精华帖子、发布圈内公告、管理圈内成员；

4. **创建圈子**：普通用户满足注册7天、发布5条以上动态即可申请创建圈子，运营后台审核通过后上线。

### 3\.1\.6 关注与好友私信模块

1. 用户可关注其他用户，查看关注列表与粉丝列表；

2. 支持一对一实时私信聊天，基于WebSocket实现消息已读、未读、实时推送；支持发送文字、图片、表情包；

3. 消息通知中心：统一收纳点赞提醒、评论提醒、私信提醒、圈子公告提醒、系统公告提醒，未读消息红点数字提示。

### 3\.1\.7 搜索与举报模块

1. 全局搜索：支持搜索用户昵称、动态内容、话题标签、兴趣圈子；

2. 举报功能：针对动态、评论、私信、用户主页均可发起举报，举报类型包含：低俗色情、广告引流、人身攻击、违法内容、抄袭搬运；举报后后台自动接收工单，管理员限时处理。

## 3\.2 圈子管理员功能模块

1. 圈子成员管理：审核入圈申请、移出违规圈友、设置圈内黑名单；

2. 圈内内容管理：删除圈内违规动态和评论、置顶优质内容、加精优质帖子；

3. 圈子设置：修改圈子头像、简介、入圈规则、圈内公告；

4. 查看圈子数据：圈内日活、发帖量、互动量基础数据统计。

## 3\.3 平台管理后台功能模块（PC端）

### 3\.3\.1 用户管理

查看全站用户列表、搜索用户、查看用户详情；支持手动封禁/解封用户（限时封禁、永久封禁）；重置用户密码；查看用户发帖、互动记录；批量清理僵尸账号。

### 3\.3\.2 内容审核管理

1. 待审核内容列表：机器风控拦截的可疑动态、评论，人工复审通过/驳回；驳回需填写驳回原因，同步推送至用户消息中心；

2. 已发布内容管理：后台可删除全站任意违规动态、评论；

3. 敏感词库配置：后台自定义添加/删除敏感词，支持批量导入敏感词，实时生效拦截违规文案。

### 3\.3\.3 圈子管理

审核用户创建圈子的申请；编辑圈子信息、下架违规圈子、转让圈主权限；划分圈子分类、配置圈子首页推荐位。

### 3\.3\.4 举报工单管理

接收全站所有举报工单，查看举报内容、举报人与被举报人信息；处理工单：驳回举报、删除违规内容、处罚违规用户；工单处理完成后同步反馈结果给举报人。

### 3\.3\.5 系统配置管理

发布全站系统公告；配置首页轮播图；设置平台基础规则（每日发帖上限、图片大小限制、视频时长限制）；管理后台操作员账号与权限。

### 3\.3\.6 数据统计大屏

可视化展示平台核心运营数据：每日注册用户数、日活/月活用户、动态发布总量、评论互动总量、圈子活跃度、举报工单数量；支持数据按日/周/月导出Excel报表。

## 3\.4 消息实时推送能力（核心技术需求）

平台所有即时互动功能必须基于WebSocket长连接实现，禁止轮询刷新，包含以下实时场景：

- 帖子新评论、新回复实时推送

- 点赞、收藏、关注行为实时消息提醒

- 私信聊天消息实时收发、已读状态同步

- 系统公告、圈子公告全员实时推送

# 4 非功能性需求

## 4\.1 性能需求

1. 页面首屏加载时间：Web端≤1\.5s，H5端≤2s；

2. 接口响应时间：普通业务接口≤300ms，实时消息接口≤100ms，**AI相关接口响应≤800ms**（AI生成文案/评论允许合理时延，添加加载动画缓冲）；

3. 并发支撑：单服务器支持同时在线2000人实时互动，支持500人同时发帖，支持**100路并发AI请求**；

4. 图片/视频压缩：前端自动压缩上传资源，单张图片≤5MB，短视频≤60s/20MB，所有媒体文件统一存入本地MinIO对象存储；

5. AI缓存优化：高频AI请求结果存入Redis缓存，减少重复调用大模型，降低服务器算力消耗。

## 4\.2 兼容性需求

1. Web端：兼容Chrome、Edge、Firefox最新版浏览器，兼容360浏览器极速模式；

2. H5端：兼容iOS、安卓全主流手机系统，适配移动端所有屏幕尺寸；

3. 管理后台：仅适配PC端1920\*1080及以上分辨率显示器。

## 4\.3 安全需求

1. 数据传输：全站HTTPS加密传输，防止数据劫持；

2. 接口安全：所有用户端接口增加Token身份校验，防止非法请求，AI接口额外增加接口签名防刷；

3. 防刷机制：限制单用户每日发帖、评论、私信次数，同时**限制单用户每日AI调用次数（每日上限50次）**，防止恶意刷取AI资源；

4. 数据安全：用户密码采用**Bcrypt自适应加盐加密存储**，无任何MD5遗留加密逻辑；敏感用户信息脱敏展示；

5. AI数据安全：私有化本地大模型全程内网调用，用户文案、图片、聊天数据**绝不外传公网**，贴合本地私有化部署要求；AI生成内容留存日志仅管理员可见。

## 4\.4 易用性需求

1. 界面风格简洁年轻化，贴合兴趣社交产品调性，操作流程不超过3步完成核心发帖、互动操作；

2. 所有报错提示采用通俗中文提示，禁止展示后端代码报错信息；

3. 移动端H5适配手势操作，支持下拉刷新、上拉加载；
