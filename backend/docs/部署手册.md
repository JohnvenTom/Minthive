# MintHive 兴趣社交平台 部署手册

> 版本：1.0.0
> 适用环境：开发 / 生产

## 1. 环境要求

| 组件 | 版本要求 | 说明 |
|------|----------|------|
| JDK | 17+ | 必须 |
| Maven | 3.8+ | 构建工具 |
| MySQL | 8.0+ | 主数据库 |
| Redis | 6.0+ | 缓存/向量存储 |
| MinIO | 最新稳定版 | 本地对象存储 |
| Ollama | 最新版 | 本地大模型服务(可选) |

## 2. 中间件部署

### 2.1 MySQL 部署

1. 安装 MySQL 8.0
2. 执行初始化脚本：
```bash
mysql -uroot -p < docs/sql/init.sql
```
3. 验证数据库 `minthive` 创建成功，包含 11 张表

### 2.2 Redis 部署

1. 安装 Redis 6.0+
2. 启动 Redis 服务：
```bash
redis-server --port 6379
```
3. 生产环境建议配置密码

### 2.3 MinIO 部署

1. 下载 MinIO 服务端：https://min.io/download
2. 启动 MinIO：
```bash
minio server /data/minio --console-address ":9001"
```
3. 默认账号：`minioadmin / minioadmin`
4. 访问控制台：http://localhost:9001
5. 创建存储桶 `minthive`(应用启动时会自动创建)

### 2.4 Ollama 本地大模型部署(可选)

1. 安装 Ollama：https://ollama.com
2. 拉取 Qwen3-7B 模型：
```bash
ollama pull qwen3:7b
```
3. 启动 Ollama 服务(默认端口 11434)：
```bash
ollama serve
```
4. 验证：访问 http://localhost:11434

## 3. 后端部署

### 3.1 配置文件

#### 开发环境
修改 `src/main/resources/application-dev.yml`：
- MySQL 连接信息
- Redis 连接信息
- MinIO 配置
- JWT 密钥
- AI 模式选择(cloud/local)

#### 生产环境
修改 `src/main/resources/application-prod.yml` 或通过环境变量注入：
- `MYSQL_USER` / `MYSQL_PASSWORD`
- `REDIS_HOST` / `REDIS_PASSWORD`
- `MINIO_ENDPOINT` / `MINIO_ACCESS_KEY` / `MINIO_SECRET_KEY`
- `JWT_SECRET`
- `AI_MODE` (cloud / local)

### 3.2 构建打包

```bash
cd backend
mvn clean package -DskipTests
```

构建产物：`target/minthive-backend.jar`

### 3.3 启动服务

#### 开发环境
```bash
mvn spring-boot:run
```

#### 生产环境
```bash
java -jar -Xms512m -Xmx2g \
  -Dspring.profiles.active=prod \
  minthive-backend.jar
```

### 3.4 验证启动

1. 访问 http://localhost:8080 验证服务启动
2. 访问 http://localhost:8080/doc.html 查看接口文档
3. 默认管理员账号：`admin / admin123`(需通过注册接口重置密码)

## 4. AI 双模式切换

### 4.1 公有云模式(cloud)
```yaml
ai:
  mode: cloud
  cloud:
    base-url: https://api.openai.com
    api-key: sk-xxx
    model: gpt-4o-mini
```

### 4.2 本地私有化模式(local)
```yaml
ai:
  mode: local
  local:
    base-url: http://localhost:11434
    model: qwen3:7b
```

切换模式仅需修改 `ai.mode` 配置项，无需改动代码，重启服务生效。

## 5. 敏感词库管理

### 5.1 默认词库
位于 `src/main/resources/sensitive-words.txt`，每行一个敏感词。

### 5.2 外部词库
通过 `sensitive.word.external-path` 配置外部词库路径，优先级高于默认词库。

### 5.3 批量导入
调用管理后台接口 `POST /api/admin/config/sensitive/import` 上传词库文件。

## 6. 防刷限流配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `ai.rate-limit.daily` | 50 | 每用户每日AI调用上限 |
| `rate-limit.post-daily` | 50 | 每用户每日发帖上限 |
| `rate-limit.comment-daily` | 200 | 每用户每日评论上限 |
| `rate-limit.message-daily` | 100 | 每用户每日私信上限 |

## 7. 文件上传限制

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `file.image-max-size` | 5 | 单张图片最大MB |
| `file.video-max-size` | 20 | 视频最大MB |
| `file.video-max-duration` | 60 | 视频最大时长(秒) |

## 8. 常见问题

### 8.1 启动报错：无法连接 MySQL
- 检查 MySQL 服务是否启动
- 检查 `application-dev.yml` 中连接信息
- 确认数据库 `minthive` 已创建

### 8.2 启动报错：无法连接 Redis
- 检查 Redis 服务是否启动
- 检查密码配置

### 8.3 文件上传失败
- 检查 MinIO 服务是否启动
- 检查存储桶权限(建议设为 public-read)
- 检查 `minio.endpoint` 配置

### 8.4 AI 接口返回降级文案
- 检查 AI 模式配置(cloud/local)
- 公有云模式：检查 API Key 是否有效
- 本地模式：检查 Ollama 服务是否启动、模型是否已拉取
- 检查每日调用次数是否超限(默认 50 次)

### 8.5 WebSocket 连接失败
- 确认 Token 有效
- 检查 `/ws/**` 路径是否被拦截器放行
- 生产环境需配置 HTTPS + wss

## 9. 日志配置

- 开发环境：控制台输出 DEBUG 级别日志
- 生产环境：输出到 `/opt/minthive/logs/minthive.log`，级别 INFO

## 10. 性能建议

- 生产环境 MySQL 连接池建议 50+ 连接
- Redis 连接池建议 64+ 连接
- JVM 堆内存建议 2GB+
- 单服务器支持 2000 人同时在线、500 人并发发帖、100 路并发 AI 请求
